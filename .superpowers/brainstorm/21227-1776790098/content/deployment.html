How do you want to deploy and host the application?

This affects architecture decisions and ongoing costs

A

Free Tier / Low Cost (Recommended for Portfolio)

Frontend: GitHub Pages, Vercel, or Netlify (free)
Backend: Render.com or Railway (free tier)
DB: MongoDB Atlas (free tier - 512MB)
AI Calls: Pay-per-use (Claude/OpenAI APIs)
Cost: ~$0-10/month depending on usage

B

AWS (Enterprise-Grade)

Services: EC2/ECS, S3, CloudFront
DB: MongoDB Atlas or DynamoDB
Benefits: Showcases AWS skills (relevant to current role)
Cost: ~$20-50/month (can use free tier initially)

C

Hybrid (Static + Serverless)

Frontend: Static hosting (GitHub Pages/Vercel)
Backend: Serverless functions (Vercel/Netlify)
DB: MongoDB Atlas or Supabase
Benefits: Cost-effective, auto-scaling
Cost: Near-zero for low traffic

D

Simple & Fast (Static Site with Client-Side AI)

Approach: Single-page Angular app
Hosting: GitHub Pages (free)
AI: Client-side API calls (user provides their API key)
Benefits: Fastest to deploy, zero hosting cost
Limitation: No backend features, no data persistence
